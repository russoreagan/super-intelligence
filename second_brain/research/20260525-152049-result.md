# Research: Note Chart Creation — Prior Art

Survey of how others have approached automatic note chart generation for rhythm games (DDR/StepMania, osu!, Clone Hero/Guitar Hero, Taiko, Beat Saber-adjacent).

## 1. The canonical two-stage decomposition

Almost every published approach splits the problem into two subtasks:

1. **Step placement / onset selection** — *when* should a note occur? (binary classification per audio frame)
2. **Step / pattern selection** — *which* note(s) should occur at that moment? (lane, color, chord, arrow direction)

This decomposition was popularized by **Dance Dance Convolution (DDC)** (Donahue, Lipton, McAuley, ICML 2017, arXiv:1703.06891). DDC uses:
- A **CNN + RNN** for step placement on mel-spectrogram frames, conditioned on a target difficulty.
- A **conditional LSTM** generative model for step selection that emits the arrow sequence given the chosen onset times.

Reported quality: charts roughly as playable as human-choreographed ones. The code is reproduced at github.com/CallumJHays/dance-dance-convolution.

## 2. Subsequent ML approaches

- **TaikoNation** (arXiv:2107.12506) — multi-layer LSTM for Taiko no Tatsujin; explicitly biases toward congruent patterns mirroring musical motifs rather than just frame-level onsets.
- **AutoOsu** (ISMIR 2023) — CRNN conditioned on a target difficulty, generates osu! charts end-to-end from audio.
- **osumapper** (kotritrona, GitHub) — TensorFlow-based beatmap generator for osu!, two-stage similar to DDC.
- **Mapperatorinator** (OliBomby, GitHub) — newer AI framework operating directly on spectrograms; supports all osu! gamemodes and modding existing maps. Aligns timing by listening to onsets rather than trusting human-provided offsets.
- **Beat-Aligned Spectrogram-to-Sequence** (arXiv:2311.13687) — reframes chart generation as conditional sequence generation (transformer) over beat-aligned segments, sidestepping the binary class-imbalance problem of frame-level onset classification.
- **Dance Dance ConvLSTM** (arXiv:2507.01644) — recent variant improving on DDC's temporal modeling.
- **Udo et al.** — DNN trained on StepMania data predicts a *high-density* chart, then a refinement filter sparsifies it to a target difficulty using a reference targets-per-measure profile. Decouples "where notes could go" from "where they should go at this skill level."
- **Shu Nakamura** (Medium) — small-scale "automatic drummer" CNN, useful as a starter reference for the placement subnet.
- **AutoRhythm** (Garlov / woosle1234, GitHub) — open-source rhythm game that auto-maps from raw frequency content; non-ML baseline but good for comparison.

## 3. Non-ML / DSP baselines still used in production

- **Onset detection** via spectral flux, complex-domain detection, or superflux (librosa, aubio, madmom) → quantize to nearest beat → emit a note. This is the floor that all ML approaches must beat.
- **Tempo / beat tracking** (madmom DBN beat tracker, librosa.beat.beat_track) before placement, so notes snap to grid.
- **Source separation** (Demucs, Spleeter) to chart per-instrument: extract drums → chart drums; extract vocals → chart vocal melody. This is the standard preprocessing step for Clone Hero drum charts now.
- **MIDI-driven charting** — when a MIDI exists, skip audio analysis entirely. The community **Auto-Chart Engine** (PyPI: `auto-chart-engine`) generates Clone Hero drum charts from MIDI by isolating the drum track and mapping General MIDI drum notes to pad colors.
- **Pitch-bucket guitar charting** — naive but common heuristic: low pitches → green, high pitches → orange. Still the starting point in many community tools.

## 4. Community authoring tools (manual / semi-automatic)

- **Moonscraper Chart Editor** (FireFox2000000, GitHub) — the de facto manual editor for Clone Hero / GH / Rock Band style `.chart` and `.mid` files.
- **osu! Beatmap editor + AiMod** — built-in "AI" mod that flags timing/spacing problems in human-authored maps. More of a linter than a generator.
- **Fullcombo.net Autocharter** and various forum tools — semi-automated guitar/drum charters built on the MIDI + pitch-bucket approach.

## 5. Recurring design lessons

- **Class imbalance is the hardest problem.** Most audio frames contain *no* note. Frame-level binary classification collapses to "predict no note" unless you weight loss, balance batches, or reframe as sequence generation (the Beat-Aligned Spectrogram paper's motivation).
- **Difficulty conditioning matters.** Either condition the generator on a difficulty embedding (DDC, AutoOsu) or generate dense then sparsify with a filter (Udo et al.). Without it, you produce one fixed difficulty.
- **Beat-alignment beats raw-time prediction.** Quantizing predictions to a beat grid (or training on beat-aligned segments) dramatically improves "feel" — humans are extremely sensitive to off-grid notes.
- **Patterning > per-note accuracy.** TaikoNation's contribution: locally-accurate onsets still feel bad if they don't form recognizable patterns. Sequence models that see context help.
- **Spectrograms are the standard input.** Mel-spectrogram with ~10ms frames is the consensus representation. Some recent work (Mapperatorinator) uses raw spectrograms with transformers to let the model learn its own features.
- **Source-separate first when charting a specific instrument.** Drum charts from full mixes are much worse than drum charts from Demucs-separated drum stems.

## 6. Suggested pipeline for a new charter

If building from scratch today, the lowest-risk path is:

1. **Preprocess**: Demucs (if instrument-specific) → mel-spectrogram, beat-track with madmom.
2. **Placement**: CNN-RNN on spectrogram frames, conditioned on difficulty (DDC-style). Or a transformer over beat-aligned windows (newer style).
3. **Selection**: conditional LSTM/transformer emitting lane/color sequences given placement.
4. **Post-process**: snap to beat grid, apply playability constraints (no impossible chords, minimum spacing per difficulty).
5. **Evaluation**: hold out human charts; measure F1 on placement, perplexity on selection, and run human playability A/B.
